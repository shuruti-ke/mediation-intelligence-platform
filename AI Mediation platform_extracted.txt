Proposed Name: Mediation Focus AI Hub
Founder: Norah Njeri Ongwenyi, MBA
IMI Qualified Mediator | MAC Accredited Mediator
Website: www.mediationfocus.co.ke

1️⃣ PLATFORM ARCHITECTURE OVERVIEW
The platform will consist of three integrated ecosystems:
Mediator Portal (Professional Dashboard)
Client / Public Portal
Admin Dashboard (System Control & AI Engine Management)
The system must support secure interaction between the portals while maintaining strict confidentiality and role-based access control.

G. Online Mediation Operations Suite

1️⃣ Case Management System (CMS)
The CMS must be a secure, end-to-end workflow system managing mediation cases from intake to closure.
A. Case Intake & Registration
Secure online intake form
Dispute category classification
Automated case number generation
Upload of preliminary documents
Conflict of interest declaration prompt

B. Case Allocation System
The CMS must support structured and transparent allocation mechanisms:
i. Court-Annexed Allocation
Case assignment notification
Rotation-based allocation tracking
Conflict-of-interest declaration workflow
Recusal and reassignment functionality
ii. Private Mediation Allocation
Client-selected mediator option
Admin-assisted assignment
AI-assisted matching (based on expertise, availability, language, industry)
Case complexity classification
The system should maintain:
Allocation audit trail
Time-stamped assignment records
Transparency logs

C. Case Progress Tracking
Status flow (Intake → Assigned → Active → Settlement → Closed)
Timeline monitoring
Session scheduling integration
Automated reminders
Milestone tracking

D. Case Summary Upload & Reporting
The CMS must allow:
Secure upload of:
Mediator’s summary report
Settlement agreement
Non-settlement report
Structured case summary template (standardised format)
AI-assisted case summarisation tool
Version tracking of uploaded reports
Secure archive storage
Restricted access permissions
Optional advanced feature:
Exportable summary in Judiciary-compliant format
Redacted version generator (for training purposes)

E. Data Security & Confidentiality Controls
Role-based file access
Encrypted storage
Two-factor authentication
Audit log of file downloads and edits
